# モンテカルロシミュレーションコード群 (monte_simulations_ai)

核融合周辺プラズマにおける中性粒子輸送・挙動解析用のモンテカルロシミュレーションコード群。
各ディレクトリは異なる次元モデルや物理的アプローチ（非アナログ法、イベント駆動、フィードバック等）で実装されている。

## シミュレーションモデル一覧

### 0次元モデル
* **`0D3V_bugver`**
  * **概要**: 0D3Vモデル。衝突による速度分布の緩和過程を計算。
  * **注意**: バグ・古いパラメータあり（過去の断面積近似式など）。

### 1次元モデル
* **`1D3V`**
  * **概要**: 1D3Vの標準中性粒子輸送モデル。CX, EL, EIを考慮。
  * **実行**: `make` 後、`run/input.nml` で設定し `./monte_carlo` を実行。
* **`1D3V_flux`**
  * **概要**: `1D3V` ベース。一定フラックスで粒子が境界から入射するモデル。
* **`1D3V_one_particle`**
  * **概要**: 特定の1粒子を追跡し、衝突頻度やエネルギー遷移のヒストグラムを取得する検証用コード。

### 3次元モデル（座標は使用していないので実質０次元）
* **`3d3v_natl`**
  * **概要**: 3D3Vの非アナログ・トラック長 (NATL) 法シミュレーション。OpenMP対応。
* **`3d3v_event-driven`**
  * **概要**: リファクタリングによるイベント駆動型モデル。計算効率が向上。最新の断面積計算式を適用。
* **`3d3v_feedback`**
  * **概要**: 粒子エネルギー沈着によるプラズマ温度低下（フィードバック機構）を考慮したモデル(単一温度 $T_i = T_e = T$)。
* **`3d3v_lookup`**
  * **概要**: 修正した断面積をルックアップテーブルとして使用するモデル。trの集計タイミングも実衝突時に変更

### アンサンブル・テスト用
* **`3d3v_repeat`**
  * **概要**: 繰り返し実行とアンサンブル統計処理用（`seiri.py` 等が含まれる）。
* **`test`**
  * **概要**: CDF断面積データの読み込み・作図等の独立テストコード (`test_dd_00` 等)。
  * **`cdf_cx-el.py`**: CDF断面積データの修正（ELからCXを差し引く）

## 開発の系譜
概ね以下の順序で発展・改修されている：
`0D3V_bugver` → `1D3V` 系 → `3d3v_natl` → `3d3v_event-driven` 
→ `3d3v_feedback` → `3d3v_lookup`

## 一般的な使用方法
1. 各ディレクトリで `make` を実行してコンパイル。
2. `input.nml` や `histogram_input.nml` でパラメータを設定。
3. 生成された実行ファイルを実行。
詳細は各ディレクトリにある `README` などを参照のこと。

## 断面積データの修正について
現在使用しているdd_00_elastic.cdf (Bachman(1995))のデータが荷電交換を含んでしまっていることが判明している。

### `dd_00_elastic.cdf` のデータ構造とエスティメータによる違い
CDFファイルの最後尾にある1次元配列 `xs_data_tab` には、以下の順序で配列データが格納されている（すべて連続した1次元配列として並んでいる）。

1. `cross_section` (101個)
2. `reaction_rate` (2601個)
3. `I_1_0` (2601個)
4. `I_1_1*up` (2601個)
5. `I_1_2*up^2` (2601個)
6. `scattering_angle` (12801個)
7. `sigv_max` (1個)
8. `angle_min` (1個)

モンテカルロのエスティメータ（推定法）によって、参照するデータが異なる点に注意が必要である。

* **Collision Estimator (`ntvel_el.f` 等)**
  * **使用データ**: `cross_section`, `scattering_angle` などを直接読んでいる。
  * **特徴**: 実質的な散乱イベント発生確率と、その際の散乱角を個別にモンテカルロサンプリングして期待値を計算する。そのため、断面積データを変更すると直接的にスコアに影響する。

* **Track-length Estimator (`ntsctr_el.f` 等)**
  * **使用データ**: `I_1_0`, `I_1_1*up`, `I_1_2*up^2` (各2601個の配列)。
  * **特徴**: 実行時の計算コストを下げるため、**あらかじめ運動量・エネルギーの移行量（輸送断面積 $\int (1-\cos\theta)\frac{d\sigma}{d\Omega} d\Omega$ など）をプラズマ粒子分布で重み付き積分したテーブルデータ(`I_1_x`)** を利用して期待値を計算する。
  * **注意点**: もしPython等でCDFの `cross_section` や `scattering_angle` のみを手動で書き換えた場合、これらの重み付き積分テーブル (`I_1_x`系) が更新されていないと、Track-length estimator のスコアには一切変更が反映されない。断面積を修正する際は、対応する積分テーブルも再計算して上書きする必要がある。
